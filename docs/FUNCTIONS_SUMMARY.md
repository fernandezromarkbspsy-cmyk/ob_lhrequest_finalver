## Project Purpose

This project is a Go/Echo linehaul dashboard with server-rendered HTML templates, GORM/Postgres persistence, and vanilla JavaScript for client-side request tables, modals, filtering, notifications, and role-specific actions.

## Main Workflow

- Ops PIC and FTE Ops create linehaul requests.
- New requests start as Pending (PENDING_OPS) and appear across request views.
- FTE Ops can edit, cancel, or approve requests.
- Approved requests route to FTE MM as Approved (PENDING_MM).
- FTE MM can reject back to FTE Ops, assign the truck, then assign plate details to move the request to For Docking.
- Dock Officer / Doc Officer handles For Docking requests by entering driver ID, LHTrip number, and docking time.
- Queue counts, notifications, and table statuses are shared across roles through /api/stats and /api/requests.

## Runtime Entry Points

- cmd/server/main.go: application startup, DB connection, GORM migration, workflow status constraint migration, Echo setup, static files, route registration, and template loading.
- internal/routes/routes.go: HTTP route map.
- internal/handlers/dashboard.go: page handlers, API handlers, request lifecycle, stats, filtering, and response shaping.
- web/static/app.js: browser behavior for login, role visibility, request modals, action modals, tables, notifications, CSV export, and theme/sidebar controls.

## Status Values

- PENDING_OPS: shown as Pending; FTE Ops queue.
- PENDING_MM: shown as Approved; FTE MM queue.
- ASSIGNED: shown as Assigned; plate assignment step.
- FOR_DOCKING: shown as For Docking; Dock Officer queue.
- CONFIRMED: retained for compatibility with existing data.
- REJECTED: rejected by FTE MM and routed back to FTE Ops.
- CANCELED: canceled by FTE Ops.
